# 밸런스 테스트 가이드

## 목적

`npm run test:balance`는 일반 캐릭터의 `maxHp`, `speed`, `attackPower`, `baseAttackRange`를 기준으로 모든 1:1 조합을 자동 대전합니다. 같은 시드는 언제나 같은 결과를 내므로, 수치 변경 전후의 의도하지 않은 기본 스탯 회귀를 빠르게 확인할 수 있습니다.

## 실행

```bash
npm run test:balance
```

기본값은 시드 `20260713`, 상대 조합당 25회입니다. 스크립트는 각 캐릭터가 모든 다른 캐릭터와 같은 횟수로 대전했는지와, 같은 시드에서 결과가 재현되는지를 함께 검증합니다.

## 해석 원칙

- 이 결과는 **기본 공격 기준선**입니다. 캐릭터별 스킬, CC, 소환물, 팀 목표, 보스 패턴은 포함하지 않습니다.
- 패치 전후에는 같은 시드와 회차를 유지해 승률 차이만 비교합니다.
- 큰 편차가 나오면 먼저 의도된 기본 스탯 변화인지 확인하고, 실제 게임에서 스킬을 포함한 수동 플레이테스트를 수행합니다.
- 결과를 티어에 바로 반영하지 않습니다. 자동 테스트, 실제 전투 관찰, 플레이 감각을 함께 판단합니다.

## 확장 방향

스킬까지 포함한 완전 시뮬레이션은 `GameLounge`의 랜덤 소스와 시간 소스를 주입 가능하게 분리한 뒤 추가합니다. 현재 기준선 테스트는 그 전 단계로, DOM/Canvas 없이 빠르게 실행되도록 의도적으로 독립돼 있습니다.
