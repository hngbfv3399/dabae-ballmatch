# 📑 시즌 4(v4.0.0) 대공사 진행 가이드 & 작업 규칙

본 문서는 시즌 4 업그레이드 작업을 여러 번의 대화(턴)에 걸쳐 안전하고 토큰 효율적으로 수행하기 위한 진행 가이드이자 규칙 문서입니다. 
작업을 진행하는 AI 에이전트는 **반드시 이 문서를 가장 먼저 읽고 진행 상황을 업데이트**해야 합니다.

---

## 🛠️ AI 자체 작업 규칙 (토큰 절약 및 안전성 확보)

1. **상대적 코드 변경 (Replace Chunk 최소화)**:
   - `src/main.ts`는 용량이 매우 큽니다. 한 번에 전체를 수정하려고 하지 말고, 수정이 필요한 특정 UI 렌더링 함수나 리스너 부분만 타겟팅하여 부분 교체(`replace_file_content` 또는 `multi_replace_file_content`)하십시오.
2. **독립 모듈화 지향**:
   - 새로 추가되는 UI 스타일이나 독립된 비즈니스 로직(예: 7대 스탯 계산기, 스킬 트리 정의 등)은 `src/main.ts` 안에 모두 때려 넣지 말고, 별도 헬퍼 파일(예: `src/maingame/statsCalculator.ts`)을 파서 내보내기/가져오기 하십시오.
3. **스키마 인덱스 필수 준수**:
   - Convex 쿼리 작성 시 반드시 `withIndex`를 명시하고 스키마에 정의된 인덱스 구조로만 필터링하십시오. (인덱스 미지정 시 테이블 전체 스캔으로 속도 저하 발생)
4. **점진적 배포 및 빌드 체크**:
   - 한 단계를 완료할 때마다 반드시 로컬에서 `npx tsc` 또는 `npm run build`를 돌려 TypeScript 컴파일 에러를 즉시 검출하고 해결한 뒤 다음 단계로 가십시오.
5. **브라우저 사용 자제**:
   - 최종 UI 검증 목적 외에는 불필요한 브라우저 사용을 삼가고 빌드 명령을 사용하여 유효성을 검증하십시오.
6. **작업 및 진행상황 공유**:
   - 매 턴 종료 전 진행 상황 요약 보고서를 작성하여 대화 내용으로 남기고, 이 가이드의 체크리스트 진척도를 체크하십시오.
